## ユーザー側アクション

既存の kaio コンテナが稼働中の場合、イメージ内の `/room/scripts/start-training-kaio.sh` は古いままなので作り直しが必要:

```bash
spirit-room close <folder>   # 既存 kaio コンテナを停止・削除
spirit-room kaio <folder>    # 新しいイメージで再起動
```

通常モード (`spirit-room open`) には影響なし (変更はすべて kaio 系スクリプト内)。
