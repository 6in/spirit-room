**変更 1: `find_free_port()` の直後に `_docker_extra_args()` ヘルパーを追加**

`find_free_port()` 関数の閉じ括弧 `}` (line 48) の後、`# ── コマンド: open ──` コメント (line 51) の前に以下を挿入する。

**重要: 関数内の各 echo は 1 行 1 トークンで出力する。**
- `-v /var/run/docker.sock:/var/run/docker.sock` のように「フラグ + 値が 1 つの文字列」の場合は 1 行で echo
- `-e HOST_WORKSPACE=...` のように `=` で結合されている場合も 1 行で echo
- `--add-host=host.docker.internal:host-gateway` も同様に 1 行で echo
- これにより `mapfile` で読み出す際に 1 トークン = 1 配列要素となり、空白を含むパスでも単語分割が発生しない

```bash
# ── --docker フラグ用追加 docker run 引数 ──────────────────────
# D-10: --docker 指定時に docker run に追加する引数を 1 行 1 トークンで echo する
# 呼び出し側は mapfile -t で配列化すること (単語分割回避)
# 引数: $1=folder (realpath 済み絶対パス), $2=room_name
_docker_extra_args() {
    local folder="$1"
    local room_name="$2"
    local host_docker_gid
    host_docker_gid=$(stat -c '%g' /var/run/docker.sock 2>/dev/null || echo "")

    echo "-v"
    echo "/var/run/docker.sock:/var/run/docker.sock"
    echo "--add-host=host.docker.internal:host-gateway"
    echo "-e"
    echo "HOST_WORKSPACE=${folder}"
    echo "-e"
    echo "SPIRIT_ROOM_HOST_GATEWAY=host.docker.internal"
    echo "-e"
    echo "SPIRIT_ROOM_DOCKER=1"
    echo "-e"
    echo "COMPOSE_PROJECT_NAME=${room_name}"
    if [ -n "$host_docker_gid" ]; then
        echo "-e"
        echo "HOST_DOCKER_GID=${host_docker_gid}"
    fi
}
```

`mapfile -t` を使って `_docker_extra_args()` の出力 (1 行 1 トークン) を配列に格納する。これにより空白を含むパスでも単語分割が発生しない。

```bash
cmd_open() {
    # D-11: --docker はどの位置でも OK (先頭・末尾どちらでも解釈)
    local docker_flag=0
    local args=()
    for _a in "$@"; do
        case "$_a" in
            --docker) docker_flag=1 ;;
            *) args+=("$_a") ;;
        esac
    done
    local folder="${args[0]:-$(pwd)}"
    folder=$(realpath "$folder")
    local name=$(folder_to_name "$folder")
    local port=$(find_free_port)

    if docker ps --format '{{.Names}}' | grep -q "^${name}$"; then
        echo "[INFO] 部屋 '$name' はすでに起動中です"
        echo "[INFO] 入室: spirit-room enter $folder"
        return
    fi

    echo "
╔══════════════════════════════════════════════╗
║  部屋を開く                                  ║
║  フォルダ : $folder
║  コンテナ : $name
║  SSHポート: $port
╚══════════════════════════════════════════════╝
"

    # D-19: --docker モード時のセキュリティ警告 (確認プロンプトなし)
    if [ "$docker_flag" = "1" ]; then
        echo "[WARN] --docker モード: /var/run/docker.sock をマウントしました。"
        echo "       コンテナ内から host の全 docker コンテナ操作が可能 = ホスト root 相当の権限です。"
    fi

    # 認証ボリュームを先に作成（なければ）
    docker volume create "$AUTH_VOLUME"         > /dev/null 2>&1 || true
    docker volume create "$OPENCODE_AUTH_VOLUME" > /dev/null 2>&1 || true

    # D-10: --docker 時の追加引数を mapfile で配列化 (1 行 1 トークン形式で空白安全)
    local _extra_args=()
    if [ "$docker_flag" = "1" ]; then
        mapfile -t _extra_args < <(_docker_extra_args "$folder" "$name")
    fi

    docker run -d \
        --name "$name" \
        --hostname "$name" \
        -e ROOM_NAME="$(basename "$folder")" \
        -e HOST_UID="$(id -u)" \
        -e HOST_GID="$(id -g)" \
        "${_extra_args[@]}" \
        -p "${port}:22" \
        -v "${folder}:/workspace" \
        -v "${AUTH_VOLUME}:/root/.claude" \
        -v "${OPENCODE_AUTH_VOLUME}:/root/.config/opencode" \
        -v "${HOME}/.claude.json:/root/.claude.json:ro" \
        "$BASE_IMAGE"

    echo "[INFO] 部屋を開きました"
    echo ""
    echo "  入室 : spirit-room enter $folder"
    echo "  または: ssh goku@localhost -p $port  (パス: spiritroom)"
    echo "  確認 : spirit-room list"
```

```bash
cmd_kaio() {
    # D-11/D-13: --docker はどの位置でも OK。kaio 固有のロジックには影響しない
    local docker_flag=0
    local args=()

    done

    local name="$(folder_to_name "$folder")-kaio"

        echo "[INFO] 界王星 '$name' はすでに起動中 — 入室します"
        cmd_enter "$folder"
        return
    fi

    echo "
╔══════════════════════════════════════════════╗
║  界王星を開く (重力10倍)                    ║
║  フォルダ : $folder
║  コンテナ : $name
║  SSHポート: $port
║  モード   : GSD 駆動・本格開発
╚══════════════════════════════════════════════╝
"

    # D-19: --docker モード時のセキュリティ警告
    if [ "$docker_flag" = "1" ]; then
        echo "[WARN] --docker モード: /var/run/docker.sock をマウントしました。"

    docker volume create "$AUTH_VOLUME" > /dev/null 2>&1 || true

    # ホスト側で refresh 済みのトークンを常にボリュームに反映する。

    local host_creds="${HOME}/.claude/.credentials.json"

        echo "[INFO] ホストの最新認証情報をボリュームに同期中..."

            -v "${host_creds}:/src/.credentials.json:ro" \
            -e HOST_UID="$(id -u)" \
            -e HOST_GID="$(id -g)" \
            --entrypoint /bin/bash \
            "$BASE_IMAGE" \
            -c "cp /src/.credentials.json /dst/.credentials.json && chown \$HOST_UID:\$HOST_GID /dst/.credentials.json && chmod 600 /dst/.credentials.json" \

        echo "[WARN] ホストに認証情報なし: $host_creds"
        echo "       spirit-room auth を先に実行してください"
    fi

    # D-10/D-13: --docker 時の追加引数を mapfile で配列化 (認証同期の --rm docker run には含めない)
    local _extra_args=()
    if [ "$docker_flag" = "1" ]; then
        mapfile -t _extra_args < <(_docker_extra_args "$folder" "$name")
    fi

    docker run -d \
        --name "$name" \
        --hostname "$name" \
        -e ROOM_NAME="$(basename "$folder")" \
        -e CLAUDE_CONFIG_DIR=/workspace/.claude-home \
        -e HOST_UID="$(id -u)" \
        -e HOST_GID="$(id -g)" \
        "${_extra_args[@]}" \
        -p "${port}:22" \
        -v "${folder}:/workspace" \
        -v "${AUTH_VOLUME}:/root/.claude-shared" \
        -v "${HOME}/.claude.json:/root/.host-claude.json:ro" \
        "$BASE_IMAGE"

    echo "[INFO] 界王星を開きました"
    echo ""
    echo "  ホスト側ログ  : spirit-room logs  $folder"
    echo "  再入室        : spirit-room enter $folder"
    echo ""
    echo "[INFO] tmux セッションにアタッチ中..."
```

**変更 4 (BLOCKER 修正): メイン dispatch を `"${@:2}"` に変更**

現行の dispatch (ファイル末尾 `case "${1:-}"` ブロック) 内の `open` と `kaio` のエントリを変更する。

変更前:
```bash
    open)   cmd_open  "${2:-}" ;;
    kaio)   cmd_kaio  "${2:-}" ;;
```

変更後:
```bash
    open)   cmd_open  "${@:2}" ;;
    kaio)   cmd_kaio  "${@:2}" ;;
```

`"${@:2}"` は位置引数の 2 番目以降を全て展開する。これにより `spirit-room open --docker /path`、`spirit-room open /path --docker`、`spirit-room open --docker` (フォルダ省略) の全パターンが `cmd_open` に正しく渡される。`"${2:-}"` のままでは `--docker` か folder の片方しか渡せず D-11 の「任意の位置で --docker を許可」が実現できない。

**禁止事項:**
- `cmd_kaio` の `--rm` 認証同期 docker run に `_extra_args` を渡さない (D-10 コンテキスト: 認証同期側は sock 不要)
- `getopts` や外部パーサを使わない (D-11 Simplicity)
- `--with-siblings` 等の追加フラグを導入しない (D-17)
- 既存の `-e HOST_UID` / `-e HOST_GID` を削除したり順序を変えない
- `resolve_running_name()` / `folder_to_name()` / `find_free_port()` の実装を変更しない
- `while IFS= read -r _line; do _extra_args+=($_line); done` の引用符なし展開パターンを使わない (WARNING 修正済み: mapfile を使うこと)
