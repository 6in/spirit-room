以下の手順で SSH 経由の認証フローを実装する。

**方針：**
cmd_auth を「一時コンテナを SSH ポート付きで起動 → ユーザーに SSH コマンドを案内 → SSH セッション内で claude auth login を実行 → 完了後コンテナを自動削除」に変更する。

実装手順:
1. `spirit-room/spirit-room` の `cmd_auth` を以下のように変更:
   - 一時コンテナをバックグラウンドで起動（ポート 2299、--rm、AUTH_VOLUME マウント）
   - ユーザーに SSH コマンドを表示
   - `ssh -t root@localhost -p 2299 -o StrictHostKeyChecking=no 'claude auth login'` を実行
   - コンテナを停止

```bash
cmd_auth() {
    echo "[INFO] 認証コンテナを起動します..."
    docker volume create "$AUTH_VOLUME" > /dev/null 2>&1 || true
    local cname="spirit-room-auth-temp"
    docker rm -f "$cname" > /dev/null 2>&1 || true
    docker run -d --name "$cname" --rm \
        -p 2299:22 \
        -v "${AUTH_VOLUME}:/root/.claude" \
        "$BASE_IMAGE" > /dev/null
    sleep 2
    echo "[INFO] SSH で認証セッションを開始します..."
    ssh -t root@localhost -p 2299 \
        -o StrictHostKeyChecking=no \
        -o UserKnownHostsFile=/dev/null \
        -o LogLevel=ERROR \
        'claude auth login'
    docker stop "$cname" > /dev/null 2>&1 || true
    echo "[INFO] 認証情報をボリューム '$AUTH_VOLUME' に保存しました"
    echo "[INFO] 以降の全部屋でこの認証が使われます"
}
```

2. 動作確認:
```bash
./spirit-room/spirit-room auth
```
- OAuth URL が表示されること
- ブラウザ認証後にコードをペーストできること
- 認証完了後にコンテナが自動停止すること

3. 認証状態をボリュームで確認:

```bash
docker run --rm -v spirit-room-auth:/root/.claude -v "$HOME/.claude.json:/root/.claude.json:ro" \

```
