  <action>
以下の 2 変更を Dockerfile に適用する。

**変更 A (L52-56 の書換え):** 現行の SSH 設定ブロックを以下のとおり差し替える。

現行 (L52-56):
```dockerfile
# ── SSH設定 ─────────────────────────────────────────────────
RUN mkdir /var/run/sshd \
    && echo 'root:spiritroom' | chpasswd \
    && echo "PermitRootLogin yes" >> /etc/ssh/sshd_config \
    && echo "PasswordAuthentication yes" >> /etc/ssh/sshd_config
```

差し替え後:
```dockerfile
# ── SSH設定 ─────────────────────────────────────────────────
# root の chpasswd は残す (entrypoint で sudo 経由の管理用途に root を保持するが SSH ログインは禁止)。
# goku のパスワード設定と UID 割当は entrypoint.sh 側でランタイム処理 (HOST_UID がホストごとに異なるため)。
RUN mkdir /var/run/sshd \
    && echo 'root:spiritroom' | chpasswd \
    && sed -i 's/^#\?PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config \
    && echo "PasswordAuthentication yes" >> /etc/ssh/sshd_config
```

ポイント:
- `echo "PermitRootLogin yes" >> ...` を `sed -i 's/^#\?PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config` に変更する。Ubuntu 24.04 の `sshd_config` には既に `#PermitRootLogin prohibit-password` がコメントアウトで入っているため、`echo "..." >>` (末尾追加) では複数行混在して壊れる危険がある。`sed` で既存行を書き換える方式が安全。
- `PasswordAuthentication yes` は goku のパスワード (`spiritroom`, D-04) ログインに必要なので残す。
- `echo 'root:spiritroom' | chpasswd` は残す。root でのコンテナ内操作 (docker exec からの chown 等) と goku の sudo 経由での root 権限昇格のため。

**変更 B (sudo パッケージ有無の確認 + 必要なら追加):** 現行の L9-23 の `apt-get install -y` リストに `sudo` が含まれていない場合は追加する。

- まず現状を Read tool で確認 (`apt-get install -y` の引数リストに `sudo` という文字列があるかどうか)。
- 含まれていなければ、L20 付近の「xz-utils zip p7zip-full \」等の既存行と並べて `sudo` を追加する。配置位置は既存のアルファベット / カテゴリ順に無理やり合わせなくてよい (既存リストもカテゴリ順ではない)。具体的には、`build-essential` の直後あたりに挿入するのが自然:

現行 (例):
```dockerfile
    python3 python3-pip python3-venv pipx \
    build-essential \
    ca-certificates gnupg \
```

差し替え後 (sudo が無かった場合のみ):
```dockerfile
    python3 python3-pip python3-venv pipx \
    build-essential \
    sudo \
    ca-certificates gnupg \
```

- 既に `sudo` が含まれていれば**何もしない**。ビルドコストを増やさないこと。

**禁止事項:**
- goku ユーザーの作成・useradd・groupadd を Dockerfile に入れない (D-01 — ランタイム作成)。
- `/etc/sudoers.d/goku` の生成を Dockerfile に入れない (D-05 — ランタイム生成)。
- 既存の `# ── Git 設定 ────` ブロック (L58-64) は触らない (D-12 — root HOME 側の設定として維持)。
- 既存の WORKDIR / COPY / EXPOSE / ENTRYPOINT 等は一切変更しない。
