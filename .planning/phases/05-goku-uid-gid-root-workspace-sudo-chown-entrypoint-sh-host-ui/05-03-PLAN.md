
**重要ポイント (Pattern C3 の注意点):**
- `-e HOST_UID / -e HOST_GID` を追加する位置は `-v` の後、`--entrypoint` の前。既存の `-e` (cmd_kaio の `-e ROOM_NAME=...`) パターンと同じ位置。
- `-c "..."` 内の `\$HOST_UID` / `\$HOST_GID` は**バックスラッシュ必須**。ホスト側 bash では `$(id -u)` を展開して `-e HOST_UID=1000` のように渡すが、`-c "..."` 内の `$HOST_UID` は**コンテナ内 bash に解釈させたい**ため、ホスト側で展開されないようバックスラッシュでエスケープする。
- chown 位置は cp の後、chmod の前 (順不同だが、所有者を合わせてから権限を最終化する自然な順序)。
- `> /dev/null 2>&1 || echo ...` は既存のまま保存。

**禁止事項:**
- `-e HOST_UID=$(id -u)` のようにクォート無しにしない (既存の double quote パターンと合わせる)
- `\$HOST_UID` のバックスラッシュを忘れない (これを忘れるとホスト bash が空文字に展開して `chown :` になり失敗する)
- `chmod 600` を削らない
  </action>
